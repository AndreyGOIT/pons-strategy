# Sprint 1 – Architecture & Structure

## 1. Sprint Overview

Sprint 1 focused on establishing a structured foundation for the Pons.fi project from an architectural and professional perspective.

The goal was not to add new features, but to:

- clarify system structure
- organize documentation
- introduce professional development practices
- prepare the project for näyttö evaluation

---

## 2. Sprint Goals

- Define overall architecture
- Create documentation structure
- Establish Git workflow
- Identify strengths and weaknesses of current implementation
- Align project with näyttö criteria

---

---

### 3.5 Näyttö Alignment

Created:

- `criteria-mapping.md`

This document maps project work directly to evaluation criteria.

---

## 4. Architectural Decisions

During this sprint, the following key decisions were made:

- Separate implementation and strategy into two repositories
- Use structured documentation as primary communication tool
- Introduce ADR for documenting decisions
- Focus on clarity and maintainability over complexity

---

## 5. Challenges

- Initial lack of formal documentation
- Need to restructure thinking from “coding” to “engineering”
- Managing Git conflicts during repository setup
- Defining proper structure without over-engineering

---

## 6. Solutions

- Introduced layered documentation structure
- Used ADR to formalize decisions
- Applied Git rebase workflow to resolve conflicts cleanly
- Focused on incremental improvements instead of full rewrite

---

## 7. Näyttö Criteria Coverage

### Covered areas:

- Architecture understanding
- Documentation

- Independent problem solving

## 8. Reflection

This sprint marked a shift from implementation-focused work to a more professional engineering approach.

Key learning points:

- Importance of structure before scaling
- Value of documenting decisions
- Git workflow as part of professional competence
- Separation between “working code” and “well-designed system”

---
